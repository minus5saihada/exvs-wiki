# claudeプロンプト
exvs-wikiというサイトを作成します。
構成は下記で考えています。

1. メイン: Next.js + Payload CMS (コンテンツ管理、DB: Postgres)
   - コンテンツは運営（自分）のみが管理するため、コミュニティ編集型のwikiエンジンではなくヘッドレスCMSを採用
2. リアルタイム: PocketBase (アクセスカウンター、マッチング用)
   - マッチング機能は最終的に追加できればいい程度で今は考慮しない
   - リアルタイム閲覧者数（現在○○人がこのページを閲覧中）のためPocketBaseは残す
3. 分析: Umami (アクセス解析)
4. インフラ: Docker Compose (これらすべてを1つの docker-compose.yml で管理)
5. レンダリング: SPAは避け、SSR/SSGを基本とする（Next.js App Router + Server Components、必要な部分のみClient Component）
   - ssg x view transition api（最後に追加できるなら最後に追加でいいかも）
6. アクセスカウンター（総アクセス数、リアルタイムアクセス把握（現在○○人がこのページを閲覧中です））
7. YouTube連携: 機体詳細ページの武器欄に再生ボタンを配置し、押下でYouTubeのモーダルを表示
